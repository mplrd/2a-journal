01 · Fondations

Identité & positionnement

2A Trading Tools est un outil de tenue de journal de trading. La marque doit transmettre rigueur, lisibilité et confiance — pas la fébrilité de la spéculation, mais la discipline du suivi de performance.

Mission

Aider les traders à objectiver leur prise de décision en transformant chaque trade en donnée exploitable.

Personnalité

Sobre, technique, méthodique. Une UI dense mais lisible, façon outil de pro — proche d'un terminal financier, loin du gamification flashy.

Promesse visuelle

Lecture claire des chiffres, cohérence entre dark et light, hiérarchie typographique stricte, couleurs sémantiques utilisées uniquement pour porter du sens (gain / perte / alerte).

10 · Bonnes pratiques

Do & Don't sur le logo

Quelques règles non-négociables pour préserver la lisibilité et la cohérence de la marque.

✓ Faire
Sur fond clair neutre, taille suffisante, zone de protection respectée.
✗ Éviter
Sur un fond saturé / contrasté avec les couleurs du logo.
✗ Éviter
Étirer ou compresser le logo. Les proportions sont fixes — toujours redimensionner avec un ratio constant (CSS : aspect-ratio: 1 / 1).
✗ Éviter
Utiliser la mauvaise variante pour le fond. Sur un fond Navy, le « 2A » et la signature deviennent illisibles — basculer sur la version dark (texte crème).
Autres interdictions : recolorer le « 2A » ou la signature dans des teintes hors charte, ajouter ombres / glows / effets 3D, ajouter un contour, retirer les barres ou le « 2 », séparer les éléments, pivoter le logo, ou le surimposer sur une photo non neutre.

Design tokens

Variables CSS prêtes à coller dans un :root ou à mapper sur un tailwind.config.js.

:root {
  /* Brand */
  --brand-navy-900: #1F2A3C;
  --brand-navy-800: #283449;
  --brand-navy-700: #354560;
  --brand-green-800: #1F5E3F;
  --brand-green-700: #2D7952;
  --brand-green-500: #46926B;
  --brand-green-400: #6BAD8A;
  --brand-green-300: #9BC7AF;
  --brand-green-100: #E3F0E9;
  --brand-cream:    #E8E8E6;

  /* Semantic */
  --success: #2D7952;
  --danger:  #B5384A;
  --warning: #C98A2B;
  --info:    #2C6E8F;

  /* Neutrals */
  --gray-50:  #F7F7F5;
  --gray-100: #ECECEA;
  --gray-200: #D8D8D4;
  --gray-300: #B6B6B0;
  --gray-400: #8B8B85;
  --gray-500: #6B6E75;
  --gray-700: #3A3D44;
  --gray-900: #16181C;

  /* Typography */
  --font-sans: 'Inter', system-ui, sans-serif;
  --font-mono: 'JetBrains Mono', monospace;

  /* Radii */
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 12px;
  --radius-xl: 16px;

  /* Shadows */
  --shadow-sm: 0 1px 2px rgba(31,42,60,.06);
  --shadow-md: 0 4px 12px rgba(31,42,60,.08);
  --shadow-lg: 0 12px 32px rgba(31,42,60,.12);

  /* Spacing scale (multiples of 4) */
  --space-1: 4px;  --space-2: 8px;  --space-3: 12px;
  --space-4: 16px; --space-6: 24px; --space-8: 32px;
  --space-12: 48px; --space-16: 64px;
}
Note d'implémentation — les valeurs proposées ici partent des couleurs effectivement présentes dans les logos. Si l'application en production utilise déjà une palette légèrement différente (par exemple un vert ou un navy plus saturé), il suffit d'ajuster les tokens --brand-* ; le reste du système (échelle, sémantique, neutres) reste valide tel quel.
