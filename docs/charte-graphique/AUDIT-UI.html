01 · Cohérence avec la charte

L'app n'utilise pas vos vrais tokens de marque

Vos tokens --color-brand-* sont parfaitement définis et alignés sur le logo. Le problème : la chaîne --p-primary-* de PrimeVue (qui pilote tous les composants UI) ne pointe pas dessus.

1.1 — Le bug le plus impactant : --p-primary-color

J'ai inspecté les CSS variables de l'app en prod. Voici ce qui est défini :

/* Tokens "brand" — corrects ✅ */
--color-brand-navy-900:  #1f2a3c;  /* navy du logo */
--color-brand-green-700: #2d7952;  /* vert foncé du logo */
--color-brand-green-500: #46926b;  /* vert clair du logo */
--color-brand-cream:     #e8e8e6;  /* off-white dark mode */

/* Tokens "primary" PrimeVue — utilisés partout */
--p-primary-color: #45587a;  /* ❌ navy DÉSATURÉ — pas la marque */
--p-primary-400:   #45587a;  /* ❌ idem */
--p-primary-500:   #1f2a3c;  /* ✅ le bon navy est ici, mais inutilisé ! */
Conséquence : chaque bouton primaire, focus ring, lien primaire, highlight de l'app utilise le navy gris fade (#45587a) au lieu du navy punchy de votre logo (#1f2a3c).

Comparaison visuelle

Actuel background: #45587a

Look "neutre", proche d'un bouton secondaire. Manque de personnalité.

Proposé background: var(--brand-navy-900)

Identité immédiate. Cohérent avec le logo.

Fix

/* Dans votre theme override PrimeVue */
:root {
  --p-primary-color:          var(--color-brand-navy-900);
  --p-primary-contrast-color: var(--color-brand-cream);
  /* Ré-aligner toute la chaîne primary-50..950 sur l'échelle navy */
  --p-primary-50:  #e7eaf0;
  --p-primary-100: #c9d1de;
  --p-primary-300: #6d7c98;
  --p-primary-500: #1f2a3c;  /* déjà OK */
  --p-primary-600: #16202f;
  --p-primary-700: #0f1722;
}

1.3 — Typographie

L'app utilise Inter pour le body (cohérent avec votre charte). ✅

  • Les titres <h1> ("Tableau de bord", "Comptes"…) sont en weight 600/700 — OK ; pourrait passer en 700 plus assumé pour matcher l'esprit du logo.
  • Les valeurs numériques (104, 48.08%, -3407.51) ne sont pas en tabular-nums → les chiffres sautillent dans les colonnes (visible sur la colonne PnL des Trades).
.numeric, td.value, .kpi-value {
  font-variant-numeric: tabular-nums;
  font-family: var(--font-mono);
}

1.4 — Logo

Le logo passe correctement en cream en dark mode. ✅

  • Le logo touche le bord gauche du header → ajouter ~16-24px de padding.
  • En sidebar collapsed (rail étroit), prévoir une version monogramme « 2A » sans signature pour ne pas être tronqué.

3.3 — Action icons (edit / delete / share)

  • Taille ~16px → en dessous de la cible tactile recommandée (24-32px min).
  • En dark mode, le gris du edit disparaît ; rouge et bleu visibles.
  • Pas de tooltip systématique. Hover affiche "Modifier" sur certains, pas tous.
  • Sets différents entre Trades (edit/delete/share) et Comptes (upload/edit/delete) → pas immédiatement compréhensibles.
  • Recommandation : tooltip systématique, padding 8px (target 32×32), couleur unifiée text-muted au repos avec sémantique au hover (delete devient red).

3.4 — Filtres et inputs

  • "Filtrer par com..." → tronqué. Soit width plus large (200px min), soit label séparé au-dessus :
<label class="label">Compte</label>
<select class="input">…</select>
  • Date pickers "Du / Au" sur Performance : pas de placeholder visible, juste un cadre vide. Ajouter placeholder="JJ/MM/AAAA" ou un label "Date début".

3.5 — Calendar P&L journalier (dashboard)

Bonne idée mais sous-exploitée : couleurs trop saturées (rouge/vert flashy), pas de hover state, pas de légende sur l'intensité.

Piste d'évolution : hover sur une cellule → mini-popover avec 5 trades · 3W/2L · win 60% · +49€. Les couleurs : reprendre les verts brand au lieu des rouges/verts génériques.
04 · Dark mode

Largement fonctionnel, pas encore "soigné"

Peu de bugs visuels, mais quelques ajustements le feront passer de "fonctionne" à "designed".

4.1 — Échelle de surfaces actuellement trop plate

Le manque de séparation visuelle entre body et cards rend la hiérarchie floue en dark mode. Recommandation : adopter une échelle de surface en 3-4 niveaux.

surface-baseCanvas / body#0a0f17
surface-1Cards, table rows#131a26
surface-2Nested (modal, popover)#1c2433
surface-3Hover state, focused row#2a3346

Avec un border 1px solid rgba(255,255,255,0.06) sur les cards, le contraste devient suffisant sans monter le niveau de gris.

4.2 — Charts en dark mode

  • Le P&L cumulé en bleu vif sur fond très sombre = agressif pour les yeux.
  • Passer les courbes "neutres" en --brand-green-300 ou --brand-cream à 80% d'opacité.
  • Les axes/grilles en gris-zinc-600 (parfait en light) deviennent quasi invisibles en dark → différencier :
[data-theme="light"] .chart-axis { stroke: #6b6e75; }
[data-theme="dark"]  .chart-axis { stroke: #a1a1aa; }

4.3 — Boutons primaires invisibles

Le + Nouveau trade en dark a un fond #18181b ≈ identique au body bg. Quasi-fantôme. C'est le bug le plus visible du dark mode.

Fix immédiat : cf §3.1 — passer en background: var(--brand-cream); color: var(--brand-navy-900); en dark mode.

4.4 — Liens secondaires faiblement contrastés

"Voir tout →" sur le dashboard : barely visible en dark. Relever à --brand-green-300 ou --p-primary-200.

4.5 — Modal "Modifier la position"

  • Fond du modal ≈ fond de page → ajouter border: 1px solid rgba(255,255,255,0.1) ou drop-shadow plus marqué.
  • "Enregistrer" CTA quasi invisible (cf §4.3).
  • Inputs ont un fond plus foncé que le modal lui-même → inverser : modal surface-1, inputs surface-2.

06 · Roadmap

Ordre d'attaque suggéré

5 sprints d'1 semaine. Le 1er sprint suffit à transformer visuellement l'app.

Semaine 1 · Tokens

Override de la chaîne --p-primary-* sur le brand navy, padding logo, alignement filtres.

Effet : l'app a immédiatement l'air "2A".

Semaine 2 · Charts

Configuration ApexCharts/Chart.js avec la palette navy + verts. Adaptations dark mode (axes plus contrastés, courbes en cream).

Effet : dashboard et Performance enfin cohérents.

Semaine 3 · Composants

CTA navy/cream, badges & tags taxonomie sémantique, action icons agrandis avec tooltip systématique.

Effet : l'UI semble "designed", pas "componenté".

Semaine 4 · Empty states & hiérarchie

Empty states sur Ordres/Positions/Trades, refonte hiérarchie KPIs dashboard, micro-interactions calendar.

Effet : l'app devient "agréable" à utiliser.

Semaine 5 · Dark mode polish

Échelle de surfaces (4 niveaux), modal border, focus states, parité de contraste avec le light mode.

Effet : dark mode "premium" — pas un simple inverse du light.

Si tu cherches le « wow effect » rapide : fais juste les 3 premiers quick wins (primary color · charts · CTA) en un sprint dédié d'une journée. C'est le meilleur ROI de tout le doc.
